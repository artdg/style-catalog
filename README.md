# Exma‑style — красивый каталог офферов (демо)

UI‑аналог витрины “как `exma.io`”, но с более современным интерфейсом: категории, поиск, фильтры, избранное и тёмная тема.

Важно: проект **демонстрационный** — без реальных выплат/интеграций. Данные сейчас моковые в `src/data/offers.ts`.

## Запуск

```bash
npm install
npm run dev
```

Откройте `http://localhost:5173/`.

## SEO / размещение

- **Canonical + OG URL**: задайте `VITE_SITE_URL` в `.env` (см. `.env.example`).
- **Sitemap**: сейчас `public/sitemap.xml` со статическими страницами. Перед продом замените `https://example.com` на ваш домен и (опционально) добавьте URL офферов.

## Beget (рекомендованный прод-режим)

Ключевой момент: **LocalStorage не подходит для продакшена**. Если вы заполните контент у себя локально, посетители на хостинге этого не увидят. Поэтому для Beget добавлен **PHP API** в `public/api/` + схема БД.

### 1) База данных (MySQL)

- Создайте БД в панели Beget.
- Выполните SQL из `public/api/schema.sql`.

### 2) Конфиг и пароль админа

- Скопируйте `public/api/config.example.php` → `public/api/config.php` на сервере и заполните доступ к БД.
- Сгенерируйте хэш пароля:

```bash
php -r "echo password_hash('your-pass', PASSWORD_DEFAULT), PHP_EOL;"
```

и вставьте в `admin_password_hash`.

### 3) Деплой фронта

Локально:

```bash
npm run build
```

Загрузите содержимое папки `dist/` в `public_html/`.

### 4) ENV для фронта

Создайте `.env` при сборке (или задайте переменные в CI):

- `VITE_SITE_URL=https://ваш-домен`
- `VITE_API_BASE=/api`
- `VITE_ENABLE_ADMIN=true` (чтобы админка вообще собиралась/показывалась)

Админка защищена “MVP” логином, но **реальная защита** в этом варианте — через сервер: API использует PHP session и пароль-хэш.

## Структура

- `src/pages/*` — страницы (главная, каталог, избранное, детали)
- `src/components/*` — лейаут/карточки
- `src/data/offers.ts` — мок‑данные (то, что вы замените на API/админку)
- `src/hooks/*` — тема и избранное (LocalStorage)
